이 프로젝트의 핵심 모듈에 대한 테스트 케이스를 생성하라.

다음을 빠짐없이 커버하라:
- 정상 경로 (happy path)
- 경계값 (빈 입력, 최대값, 0, 음수, 경계 직전/직후)
- 에러 경로 (잘못된 입력, 타임아웃, 네트워크 실패, 권한 부족)
- 상태 전이 (동시 호출, 순서 의존, 멱등성)
- 호환성 (null, undefined, 빈 문자열, 빈 배열, 특수 문자)

각 테스트 케이스에 대해:
1. 테스트명 (should ... when ... 형식)
2. 사전 조건 (given)
3. 실행 (when)
4. 기대 결과 (then)
5. 카테고리 (unit / integration / e2e)
