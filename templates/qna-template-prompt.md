qna.js는 템플릿 기반 객관식 Q&A로 소프트웨어 설계 결정을 수집하는 CLI 도구다.

템플릿은 qna.js의 Claude 인터뷰어에게 "어떤 영역을, 어떤 순서로, 어떤 깊이로 질문할지" 가이드하는 문서다.

## qna.js 작동 방식

1. 사용자가 템플릿 파일을 지정하여 실행
2. Claude가 템플릿을 참고하여 객관식 질문 생성 (3-4개 선택지, 각각 이유 포함)
3. 사용자가 번호 선택 또는 직접 입력
4. 반복하며 설계 결정 누적, 영역별 커버리지 추적
5. 모든 영역 커버리지 충분 시 자동 종료
6. 결과: decisions.md (영역별 Q&A 정리)

## 좋은 템플릿의 조건

- 설계 영역을 명확히 나열 (커버리지 추적의 기준이 됨)
- 각 영역별 핵심 결정 포인트를 제시 (질문 생성의 씨앗)
- 영역 간 의존성/순서를 암시 (앞선 결정이 뒤 질문에 영향)
- 선택지의 트레이드오프 힌트 제공 (선택지 reason 생성에 활용)
- 그린필드와 브라운필드 모두 적용 가능해야 함

## 작성할 것

범용 소프트웨어 프로젝트에 적용할 수 있는 qna.js 템플릿을 작성하라.

웹 앱, API 서버, CLI 도구 등 다양한 프로젝트에 사용할 수 있되, 질문이 너무 추상적이지 않아야 한다. 실제 코드 작성에 필요한 구체적 결정을 이끌어내는 것이 목표다.

결과를 지정된 출력 파일에 마크다운으로 작성하라.
