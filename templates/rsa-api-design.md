이 프로젝트의 코드를 분석하여 API를 설계하라.

설계 시 고려할 사항:
- RESTful 원칙 (자원 중심, HTTP 메서드 의미, 상태 코드)
- 일관된 응답 형식 (성공/실패 구조, 페이지네이션, 정렬)
- 에러 처리 (에러 코드 체계, 메시지 국제화, 디버깅 정보)
- 인증/인가 (엔드포인트별 권한, API 키, 토큰)
- 버저닝 전략

각 엔드포인트에 대해:
1. 메서드 + 경로
2. 요청 파라미터/바디 (타입, 필수 여부, 제약 조건)
3. 응답 구조 (성공/실패 각각)
4. 상태 코드 목록
5. 설계 근거 (왜 이 구조인지)
