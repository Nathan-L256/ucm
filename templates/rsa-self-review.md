이 프로젝트의 최근 변경사항을 독립적으로 리뷰하라.

분석 결과와 구현된 코드를 비교하여 빠진 요구사항, 버그, 보안 이슈를 찾아라.

다음 관점에서 문제를 찾아라:
- 요구사항 충족: 분석에서 명시한 모든 항목이 구현되었는가
- 버그: 논리 오류, off-by-one, null/undefined 미처리, 경쟁 조건
- 보안: 인젝션, 인증/인가 우회, 민감 정보 노출, 입력 검증 누락
- 엣지 케이스: 빈 입력, 대용량, 동시성 문제

각 발견 항목에 대해:
1. 파일 경로와 라인 번호
2. 문제 설명 (왜 문제인지)
3. 수정 제안 (구체적 코드 포함)
4. 우선순위 (P1 — CRITICAL / P2 — IMPORTANT / P3 — MINOR)

**P1 — CRITICAL (must fix):**
- 요구사항 미충족
- 보안 취약점
- 데이터 손실 가능

**P2 — IMPORTANT (should fix):**
- 엣지 케이스 미처리
- 성능 문제

**P3 — MINOR (nice to fix):**
- 스타일 불일치
- 불필요한 코드

마지막 줄에 P1이 하나라도 있으면 `GATE: FAIL`, 없으면 `GATE: PASS`를 출력하라.
