이 프로젝트의 코드를 분석하여 마이그레이션 계획을 수립하라.

계획에 포함할 사항:
- 현재 상태 분석 (아키텍처, 의존성, 기술 스택)
- 목표 상태 정의
- 단계별 마이그레이션 절차 (각 단계가 독립 배포 가능해야 함)
- 데이터 마이그레이션 (스키마 변경, 데이터 변환, 롤백 전략)
- 호환성 유지 방안 (이전 버전과 병행 운영 기간)
- 리스크와 완화 방안
- 검증 방법 (각 단계별 성공 기준)

각 단계에 대해:
1. 단계명과 목적
2. 변경 범위 (파일, 설정, 인프라)
3. 선행 조건
4. 실행 절차
5. 롤백 절차
6. 검증 방법
