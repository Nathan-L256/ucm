이 프로젝트에서 리팩토링이 필요한 부분을 찾고 개선 방안을 제안하라.

분석 기준:
- 복잡도: 순환 복잡도가 높은 함수, 깊은 중첩, 긴 함수
- 중복: 유사한 로직이 여러 곳에 반복되는 코드
- 결합도: 모듈 간 불필요한 의존, God object, 순환 참조
- 추상화: 레이어 누락, 관심사 미분리, 인터페이스 부재
- 네이밍: 의도가 불분명한 변수/함수명, 일관성 없는 컨벤션

각 제안에 대해:
1. 대상 파일과 코드 범위
2. 현재 문제점
3. 리팩토링 방안 (구체적 구조 변경 포함)
4. 기대 효과 (가독성, 테스트 용이성, 확장성 등)
5. 리스크 (기존 동작 변경 가능성)
